# デベ向けボリュームチェック — プロダクション化設計

**日付**: 2026-03-02
**目的**: デベロッパー・不動産会社が用地取得前の初期検討 + 事業性検討資料作成に使えるレベルにする
**ターゲットエリア**: 東京23区 + 首都圏主要都市

---

## 現状の課題

1. **敷地形状の入力が面倒** — 住所だけでは敷地ポリゴンが取れない。手動入力 or PDF/画像アップが必要
2. **地区計画・高度地区が拾えない** — 用途地域は自動取得済みだが、上乗せ規制を無視 → 数字の信頼性が低い
3. **事業性計算がない** — 「最大○階○㎡」は出るが「いくらで建つのか」がわからない

## ゴール

住所を入れて地図上で敷地を選択 → 周辺の道路・水路・隣地を含む全制約を自動取得 → 最大ボリューム + 概算事業費が出る → PDF帳票で上司・クライアントに提出

---

## Phase 1: 敷地形状の自動取得 + 道路幅自動検出

### データソース

| データ | ソース | 形式 | 費用 |
|--------|--------|------|------|
| 筆界ポリゴン | AMX Project PMTiles（法務省登記所備付地図ベース） | PMTiles (MVT) | 無料 |
| 道路ネットワーク | PLATEAU tran MVT | MVT | 無料 |
| フォールバック | 既存の手動入力/PDF/画像アップロード | - | - |

### 注意: 東京の登記所備付地図カバー率

法務省データは任意座標系の区画が多く、東京都心部で**筆界ポリゴンが取れない区画がある**。
対策:
- PMTilesで取れる → そのまま使用
- 取れない → 「この区画のデータがありません。手動入力してください」と案内
- PLATEAU bldg（建物フットプリント）を参考表示 → 周辺状況の把握を補助

### アーキテクチャ

```
[住所入力] → [geocode（既存）] → [緯度経度]
  │
  ├─ AMX PMTiles → 地図上に筆界ポリゴンをオーバーレイ表示
  │   └─ ユーザーがクリック → 地番・形状を取得
  │
  ├─ PLATEAU tran MVT → 接道する道路の検出
  │   └─ 道路幅員・道路種別を自動設定
  │
  └─ フォールバック: 手動入力（既存UI）
```

### 技術選定

- **MapLibre GL JS** — PMTiles + MVT 表示。`pmtiles` npm パッケージでプロトコル登録
- **既存の @mapbox/vector-tile + pbf** — サーバーサイドでのMVT点クエリ（既に zoning-lookup で使用中）
- 新しい地図画面（敷地選択用）を追加するか、既存の SiteSection に統合するか → SiteSectionに統合（UIフロー維持）

### 新規コンポーネント / API

| 種別 | パス | 機能 |
|------|------|------|
| API | `/api/parcel-lookup` | 緯度経度→AMX PMTilesから筆界ポリゴン候補取得 |
| API | `/api/road-detect` | 緯度経度+敷地ポリゴン→PLATEAU tranから接道道路検出 |
| Component | `ParcelSelector.tsx` | MapLibre地図表示+筆界クリック選択UI |
| Hook | `useParcelSearch.ts` | 住所→筆界→道路の自動取得フロー管理 |

### AMX PMTiles 仕様

- URL: `https://habs.rad.naro.go.jp/spatial_data/amx/a.pmtiles`
- レイヤー:
  - `daihyo`（代表点）: z2-13、属性なし
  - `fude`（筆ポリゴン）: z14-16、属性あり（座標系・市区町村コード・縮尺・精度区分）
- 属性フィールド（`fude`レイヤー）:
  - 市区町村コード、市区町村名、大字名、丁目名、小字名、地番、精度区分

### PLATEAU 道路 (tran) MVT

- 東京都全域タイルURL（datacatalog APIから取得、キャッシュ）
- 道路ポリゴンから敷地ポリゴンとの接道を空間演算で検出
- 道路幅員は属性から取得（`width` フィールド）

## Phase 2: PLATEAU urf 連携（地区計画・高度地区・防火地域の自動取得）

### データソース

東京都全域の prefecture-wide MVT タイルが存在（認証不要・無料）:

| Feature Class | URL (固定) | 取得する情報 |
|--------------|------------|-------------|
| UseDistrict（用途地域） | `https://assets.cms.plateau.reearth.io/assets/5b/8d0e14-be51-4739-bf91-13cc176472c8/13_tokyo_pref_2023_citygml_1_op_urf_UseDistrict_mvt_lod1/{z}/{x}/{y}.mvt` | FAR, BCR, 用途地域コード |
| HeightControlDistrict（高度地区） | `https://assets.cms.plateau.reearth.io/assets/a2/81a1a7-03b8-4cf2-bb26-19103b32e255/13_tokyo_pref_2023_citygml_1_op_urf_HeightControlDistrict_mvt_lod1/{z}/{x}/{y}.mvt` | 高さ制限パラメータ |
| FirePreventionDistrict（防火地域） | `https://assets.cms.plateau.reearth.io/assets/d9/5ce2d6-0aa8-4a17-a86a-028c2dc2b817/13_tokyo_pref_2023_citygml_1_op_urf_FirePreventionDistrict_mvt_lod1/{z}/{x}/{y}.mvt` | 防火/準防火区分 |
| DistrictPlan（地区計画）* | ward別URLをdatacatalog APIで取得 | 地区計画の有無・名称・制限内容 |

*DistrictPlan は全wardにあるわけではない。対応wardのURLをキャッシュする。

### 現行 Geolonia タイルとの関係

| 項目 | Geolonia (現在) | PLATEAU urf (追加) |
|------|----------------|-------------------|
| 用途地域 | ✅ | ✅（クロスチェック用） |
| 容積率/建ぺい率 | ✅ | ✅（attributes JSON内） |
| 防火地域 | ✅ | ✅（別レイヤーで精度向上） |
| **高度地区** | ❌ | ✅ **NEW** |
| **地区計画** | ❌ | ✅ **NEW** |
| **高度利用地区** | ❌ | ✅ **NEW**（一部ward） |
| カバー範囲 | 全国 | PLATEAU対応都市のみ |

**戦略**: Geolonia を基本データソースとして維持。PLATEAU urf で高度地区・地区計画・防火地域を**補完**。
PLATEAU 対応都市外では Geolonia のみにフォールバック。

### アーキテクチャ

```
[既存 zoning-lookup]
  + [新規 PLATEAU urf クエリ]
  → マージして ZoningData を拡張

ZoningData 追加フィールド:
  - heightDistrict（高度地区）← PLATEAU HeightControlDistrict
  - districtPlan（地区計画）← PLATEAU DistrictPlan
  - fireDistrict 精度向上 ← PLATEAU FirePreventionDistrict
```

### 新規 API / 変更

| 種別 | パス | 機能 |
|------|------|------|
| API | `/api/plateau-urf-lookup` | 緯度経度→PLATEAU urf MVTから都市計画情報取得 |
| 変更 | `zoning-lookup/route.ts` | PLATEAU結果をマージ、ZoningData型拡張 |
| UI | `ZoningSection.tsx` | 地区計画警告表示、高度地区表示を追加 |

### 地区計画の表示方針

地区計画の詳細規制は自治体ごとにバラバラで完全自動適用は非現実的。
→ **「この敷地には地区計画があります」+ 計画名称 + 自治体HPリンク**を警告表示
→ ユーザーが手動で追加制限を入力するUIを提供

## Phase 3: 事業性計算 + PDF帳票強化

### 事業性計算

デベが用地取得判断に使う最低限の数字:

| 項目 | 入力方法 | 計算 |
|------|---------|------|
| **想定用途** | プリセット選択: マンション/オフィス/商業/ホテル/混合 | - |
| **延床面積** | 既存計算結果から自動 | maxFloorArea |
| **想定建設単価** | 用途別プリセット + カスタム入力（万円/坪） | - |
| **概算建設費** | 自動 | 延床(坪) × 建設単価 |
| **想定賃料/販売単価** | 用途別プリセット + カスタム入力 | - |
| **概算収益（年間/総額）** | 自動 | 延床(坪) × 賃料/販売単価 |
| **土地取得費** | 手動入力（億円） | - |
| **表面利回り** | 自動 | 年間収益 / (土地+建設費) |
| **事業収支概算** | 自動 | 販売総額 - (土地+建設費) |

### プリセット値（用途別デフォルト）

| 用途 | 建設単価（万円/坪） | 賃料（円/坪・月） | 販売単価（万円/坪） |
|------|-------------------|------------------|-------------------|
| マンション | 120-150 | 12,000-18,000 | 300-500 |
| オフィス | 150-200 | 15,000-25,000 | - |
| 商業 | 130-180 | 20,000-40,000 | - |
| ホテル | 200-300 | - | - |

※ エリアにより大幅に変動。プリセットはあくまで目安。カスタム入力を推奨表示。

### PDF帳票強化

現状: ブラウザ印刷（`window.print()`）のみ。計算結果の半分が未出力。

改善:
1. **事業検討レポート**として再構成
2. 既存の計算結果を全セクション出力
3. 事業性計算セクション追加
4. 3Dビュー + 日影図のスクリーンショットを埋め込み（html2canvas）

### レポート構成

```
1. 物件概要
   - 住所、地番、敷地面積、用途地域、防火地域

2. 法規制サマリー
   - 建ぺい率（指定/緩和後）、容積率（指定/前面道路）
   - 絶対高さ制限、高度地区、日影規制
   - 地区計画（あれば警告表示）

3. 最大ボリューム
   - 最大延床面積、建築面積、最高高さ、最大階数
   - 各階床面積表

4. 斜線制限詳細
   - 道路斜線、隣地斜線、北側斜線の各パラメータ

5. 日影・逆日影（計算済みの場合）
   - 建物パターン比較結果
   - 逆日影コンターライン

6. 事業性概算（Phase 3）
   - 想定用途、建設単価、概算建設費
   - 概算収益、利回り

7. 3Dビュー（スクリーンショット）

8. 免責事項
```

## 技術的な注意事項

### PMTiles 依存パッケージ追加
```bash
pnpm add pmtiles maplibre-gl
```

### PLATEAU API の不安定性
- SLAなし、スキーマ変更あり得る
- タイルURLは asset hash を含むため変更される可能性
- 対策: datacatalog API からURL取得 → サーバーサイドでキャッシュ（24h TTL）

### 登記所備付地図の東京カバレッジ問題
- 任意座標系データは位置が取れない
- 対策: PMTilesで取れない場合は明確にユーザーに伝えてフォールバック

### 既存コードへの影響
- `SiteSection.tsx` にParcelSelector統合
- `ZoningSection.tsx` に地区計画・高度地区表示追加
- `types.ts` の ZoningData 型拡張（heightDistrict, districtPlan 追加）
- `PrintReport.tsx` / `pdf-export.ts` の全面改修
